The process involved:Data Loading: Importing the train.csv dataset.Data Cleaning: Handling missing values (NaN) in columns like Gender, Married, Credit_History, etc.Feature Engineering: Converting categorical data (like Gender, Property_Area) into numerical values.Log Transformation: Applying np.log to skewed data like ApplicantIncome and LoanAmount to normalize their distribution.Training: Training a DecisionTreeClassifier on the cleaned, processed features to predict the Loan_Status column.Exporting: Saving the trained model as model.pkl using pickle.This model is loaded by the Flask app and is used by both the prediction form and the chatbot to generate the final 'Y' or 'N' prediction.üèõÔ∏è System ArchitectureThe application uses a hybrid AI approach. The process flow for the chatbot is as follows:A user logs in via Firebase Authentication.The user answers 11 questions from the chatbot.The Flask server collects all 11 answers.The server runs the preprocess_data function and feeds the results into the model.pkl (Decision Tree) to get a prediction (Y or N).The server then sends the user's data and the model's prediction to the Google Gemini API, asking it to write a user-friendly, formatted response.The final, nicely-formatted advice is sent back to the user's chat window.<img src="static/system_architecture.png" alt="System Architecture Diagram" width="100%">üñºÔ∏è ScreenshotsHome PageAbout Page<img src="static/loan.png" alt="Home Page" width="400"><img src="static/Project Activity.png" alt="Project Activity Log" width="400">Prediction FormChatbot(Image of your prediction form)(Image of your chatbot interface)LicenseThis project is licensed under the MIT License.
